# 知识库页面分层标准

## 核心原则

真正能用的知识库，不是“存了很多内容”，而是“能把新材料稳定压成可复用判断”。

页面分层要解决 4 个问题：
- 这条内容是证据，还是理解，还是判断？
- 以后我要找它时，应该按来源找，还是按主题找？
- 新材料进来后，我是新增一页，还是更新旧页？
- 这页是否能成为别的页面的稳定引用点？

## 1. 来源页

放在：
- `01 来源/`

它是什么：
- 原始材料的证据层。

应该包含：
- 标题
- 作者 / 来源主体
- 日期
- 原链接 / 原文件
- 简短摘要
- 关键摘录
- 正文或可恢复正文
- 指向相关 `02 实体/` / `03 主题/` 的链接

不应该变成：
- 主题总结页
- 个人长期观点页
- 教程大全页

判断标准：
- 如果这页高度依赖“原作者说了什么”，它通常应该是来源页。

## 2. 实体页

放在：
- `02 实体/`

它是什么：
- 人物、工具、组织、概念等相对稳定对象的累计页。

应该包含：
- 它是谁 / 是什么
- 为什么重要
- 与哪些主题相关
- 重要来源
- 新增事实或判断的增量记录

不应该变成：
- 某一篇文章的全文备份
- 单次事件报道

判断标准：
- 如果未来还会有很多来源持续指向它，就值得成为实体页。

## 3. 主题页

放在：
- `03 主题/`

它是什么：
- 某个主题的当前理解页，是“长期可更新的工作层”。

应该包含：
- 核心问题
- 子议题
- 代表来源
- 相关实体
- 当前理解
- 值得继续观察的问题

不应该变成：
- 单篇教程存档
- 单篇资源清单
- 单次爆文摘录

判断标准：
- 如果这页能回答“我目前怎么看这个主题”，它才算主题页。
- 如果它只能回答“某篇内容讲了什么”，它更像来源页。

## 4. 综合页

放在：
- `04 综合/`

它是什么：
- 跨来源后的比较、阶段结论、方法整合页。

应该包含：
- 核心结论
- 对比或整合结构
- 支持证据
- 保留意见 / 限制条件

判断标准：
- 如果一页的价值主要来自“把多个来源压成一个结论”，它适合放综合。

## 5. 主张 / 问题页

放在：
- `06 主张与问题/`

它是什么：
- 可验证判断、开放问题、冲突、踩坑的最小沉淀单元。

优先记录：
- 可复用的判断
- 明显冲突的观点
- 还没想清楚但值得持续追踪的问题
- 对工作流有长期指导价值的踩坑

判断标准：
- 如果一句话已经值得以后反复引用、验证或反驳，就不该只留在来源摘要里。

## 6. 常见误分层

这些情况最常见：
- 把“来源页”放进 `03 主题/`
- 把“资源清单”误当主题页
- 把“单篇教程”误当主题页
- 把“个人结论”写在来源页里，不单独提炼主张
- 只有来源，没有综合和主张

## 7. 快速判断问题

看到一页时先问：
1. 这页是证据，还是理解，还是判断？
2. 新材料进来后，我是更新它，还是另建一页？
3. 我以后会按“作者/来源”找它，还是按“问题/主题”找它？
4. 这页能不能成为其他页面稳定引用的锚点？

如果答不出来，这页大概率分层还不对。

## 8. 当前库里最典型的好例子

- 主题页例子：
  - `用大模型做知识管理`
  - `需求澄清与默会知识`
  - `AI 编程工具与氛围式编程`

- 综合页例子：
  - `为什么大模型 Wiki 不同于检索增强生成（RAG）`

- 主张页例子：
  - `大模型 Wiki 比朴素检索增强生成（RAG）更能复利知识`

这些页的问题不是方向错，而是数量还不够，尚未成为整个知识库的默认沉淀模式。
