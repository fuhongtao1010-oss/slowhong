# 会话阶段归档工作流

## 目的
把用户和助手在每个阶段形成的结论、方法、踩坑、配置变化，及时整理进脑子，避免重要信息只留在聊天记录里。

## 什么时候归档
- 一个任务完成后。
- 一个问题排查出根因后。
- 形成新的稳定流程、命令、配置方法后。
- 用户明确说“这个记一下 / 存脑子 / 以后按这个来”后。
- 一段会话从一个主题切到另一个主题前。

## 不要归档什么
- 寒暄、确认、低价值过程。
- 临时失败尝试，除非它能变成踩坑记录。
- 还没验证的猜测。
- 大段原始聊天全文，除非用户明确要求保留。

## 默认归档位置
- 稳定流程：`08 工作流/`
- 工具和本机方法：`07 地图/助手技能与本机说明.md` 或独立工作流页
- 排障教训：`06 主张与问题/踩坑记录/`
- 阶段性判断：`04 综合/`
- 外部资料：先建 `01 来源/`
- 临时不确定内容：`00 Inbox/`

## 每次归档格式
```md
## YYYY-MM-DD - 阶段标题

### 背景
- 这轮会话要解决什么。

### 结论
- 已确认的结果。

### 做法
- 可复用步骤、命令、路径或配置。

### 踩坑
- 发生了什么问题，根因是什么，怎么避免。

### 后续
- 还需要继续跟进什么。
```

## 执行规则
- 先完成用户当前任务，再做归档。
- 每次只沉淀有复用价值的内容。
- 归档要短，不复制聊天全过程。
- 写入后告诉用户存到了哪里。
- 如果涉及系统配置，必须记录文件路径、症状、根因、验证结果。

## 当前例子
Hermes 大模型 401 排障应归档为踩坑记录：
- 症状：`401 Missing Authentication header`
- 根因：`/home/hong/.hermes/config.yaml` 中 `smart_model_routing` YAML 缩进和重复键错误
- 影响：Hermes 解析配置失败，回退到无认证的 OpenRouter
- 修复：修正 YAML，重启 `hermes-gateway.service`
- 验证：NVIDIA `/models` 返回 `200`，`moonshotai/kimi-k2.5` 可用
