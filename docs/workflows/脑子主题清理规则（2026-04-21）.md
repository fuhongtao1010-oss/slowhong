# 脑子主题清理规则（2026-04-21）

这页只解决一个问题：
什么页面应该留在 `03 主题/`，什么页面应该移出去。

如果这个判断不稳，知识库最后一定会变成“教程、来源、清单、主题”混在一起的资料堆。

---

## 一句话标准

只有“会被持续更新的问题域”才配留在 `03 主题/`。

如果一页内容更像：
- 单篇教程
- 单次整理
- 资源清单
- 某个来源的改写版
- 一次性操作手册

那它大概率不该放在 `03 主题/`。

---

## 留在 `03 主题/` 的标准

满足越多，越像真主题：

1. 这页在回答一个会长期反复出现的问题。
2. 后续新来源进来时，你很自然会继续往这页补。
3. 这页可以挂多个来源，而不是只依赖单一来源。
4. 这页不是为了“教你怎么做一次”，而是为了形成长期理解。
5. 这页能稳定链接到相关综合页、主张页、实体页。

典型例子：
- `用大模型做知识管理`
- `代理式研究工作流`
- `AI 编程工具与氛围式编程`
- `技能生态（Skill）`
- `需求澄清与默会知识`

---

## 不该留在 `03 主题/` 的典型情况

### 1. 明显是教程
应该进：
- `08 工作流/`

例子：
- Claude Code 新手指南
- CLIProxyAPI 部署教程
- Telegram 接入教程

### 2. 明显是来源或资源清单
应该进：
- `01 来源/`

例子：
- 技能总表
- 抓取下来的榜单
- 外部网站的资源汇总

### 3. 明显是阶段性整理或跨来源结论
应该进：
- `04 综合/`

例子：
- 博主清单
- 多来源对比后的推荐
- 某个阶段的判断总结

### 4. 明显是稳定对象
应该进：
- `02 实体/`

例子：
- 人物
- 工具
- 概念
- 组织

---

## 快速判断法

看到一页时，先问 3 句：

1. 这页半年后还会继续更新吗？
2. 如果来了 3 篇新来源，我会自然把它们编到这页吗？
3. 这页的标题是在指向一个长期问题，还是指向一次性内容？

如果前两句都答不上来，基本就不是主题页。

---

## 当前库里的执行规则

以后新增页面时，默认按这个顺序判断：

1. 先问：它是不是来源？
2. 再问：它是不是实体？
3. 再问：它是不是主题？
4. 如果都不是，再看是不是综合或工作流。

不要一上来就塞进 `03 主题/`。
