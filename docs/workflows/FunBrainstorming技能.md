# Fun Brainstorming 技能

> 来源：https://github.com/roin-orca/skills/tree/main/skills/fun-brainstorming

## 是什么
一个结构化但轻量的头脑风暴技能，帮助快速从想法到可执行方向。

## 核心原则

1. **速度大于仪式** — 价值在于思考，不是产出物
2. **YAGNI** — 只设计当前需要的，不要过度设计
3. **偏向行动** — 两个选项差不多时，快速选一个开始做
4. **批量发现** — 一次性问完所有澄清问题，不要分散在多条消息里
5. **适度深度** — 任务简单就快，任务复杂才深入

## 执行流程

1. **Discover（发现）** — 了解项目背景，问最多3个聚焦问题（首选多选式）
2. **Propose（提议）** — 提出2个方案，附上利弊，直接推荐一个
3. **Converge（收敛）** — 获得用户明确批准，最多来回2轮
4. **Capture（记录）** — 记录选择的方向（是什么、为什么、关键决策）
5. **Implement（实现）** — 开始动手

## 重要规则
**不要写代码、不要创建文件，直到用户明确批准方向为止。**
