# 设计任务短指令模板

以后给 Hermes / Claude / 其他助手下设计任务，尽量直接用这种短句。

## 通用

- 先看上下文，再做，不要凭空生成。
- 先给一个可用版本，再给两个变体。
- 先读现有页面和风格，再改。

## 页面 / 网页

- 先看现有页面风格，再给我 3 个改版方向。
- 先读这个页面，再做一个更清晰的改版稿。
- 别空谈，先给一个可运行原型，再说理由。
- 先沿用现有视觉语言，不要突然换风格。

## 设计系统 / 样式

- 先总结这个产品现在的视觉语言，再补新页面。
- 先提炼颜色、字体、间距、组件风格，再出稿。
- 优先贴着现有 design system 做，不要重发明。

## 变体探索

- 先给 1 个稳妥版，再给 2 个明显不同的方向。
- 给我 3 个版本：保守版 / 品牌强化版 / 转化优先版。
- 不要只出一个方案，给我可比较的变体。

## 结果导向

- 直接给结果，不要长篇解释。
- 先出稿，再用三句话解释设计取舍。
- 先做，再说；如果上下文不够，只问会改变方向的问题。
