## 📐 万能公式

每个提示词都遵循这个结构：

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色** | 告诉AI扮演谁 | "扮演一位招聘经理" / "你是一位无情的编辑" |
| **背景** | 你的具体情况 | "我有一家12人的B2B SaaS公司，需要给企业CTO写冷邮件" |
| **任务** | 具体要做什么 | "写三个版本，每个150字以内" |
| **格式** | 输出形式 | 表格 / 编号列表 / 300字以内 |
| **约束** | 说不想要什么 | "不用行话" / "不要以'您好'开头" |

> 💡 **五要素适用于每个模型、每个任务。提示词会变，结构永远不变。**

### 秘密武器（加在任何提示词末尾）
> "在你开始之前，问我任何你需要的问题，以便给我最好的答案。"

## ✍️ 写作 (Prompts 1-5)

### 1️⃣ 邮件改写器
```
我需要给【收件人/职位】发一封邮件。情况如下：【用2-3句话描述】。语气应该是【正式/友好/直接/致歉】。字数控制在【数字】以内。不要以"希望这封邮件找到状态良好的您"开头。请写三个版本：一个直接的、一个温和的、一个正式的，由我来选择。
```

### 2️⃣ "听起来像我"改写器
```
这是我写的一段文字：【粘贴你的文字】。分析我的写作风格：句子长度、词汇水平、语气和规律。现在用我的风格（而不是你的）改写以下内容：【粘贴待改写文字】。只返回改写后的版本。
```

### 3️⃣ 一段话解释器
```
用一段话解释【概念】，让一个聪明的14岁孩子能听懂。用日常生活中的具体类比。不用行话，不要废话。如果你无法简单解释，就直说。
```

### 4️⃣ 内容钩子生成器
```
我在为【受众】写关于【主题】的内容。请写5个开头钩子，每个使用不同的技巧：
1. 一个令人惊讶的统计数据
2. 一个挑战常见认知的反直觉观点
3. 一个微故事（最多2句话）
4. 一个读者无法忽视的直接问题
5. 一个附带证明承诺的大胆声明
每个钩子最多1-2句话。
```

### 5️⃣ "删掉废话"编辑器
```
将这段文字改写，缩短40%。删掉每一个不必要的词。修正被动语态。删除任何没有新信息的句子。保持相同的含义、语气和关键要点。只返回改写后的版本：【粘贴文字】
```

## 🧠 思考 (Prompts 6-10)

### 6️⃣ 决策矩阵
```
我在【选项A】和【选项B】之间做选择。背景：【用2-3句话描述你的情况】。请建立一个对比表，包含以下列：因素、选项A、选项B、胜者。至少包含8个我应该考虑的因素。在末尾给出你的建议并解释理由。如果是势均力敌的决定，请如实说明。
```

### 7️⃣ 魔鬼代言人
```
我计划【描述你的计划/想法】。请反驳它。给我5个最有力的失败理由、我可能犯错的假设，以及我没有考虑到的风险。请直言不讳，不要软化批评。然后针对每个风险给我一个具体的应对措施。
```

### 8️⃣ "我遗漏了什么"
```
这是我对【主题/情况】的理解：【解释你知道的】。我遗漏了什么？有哪些重要因素我没有考虑到？这个领域的专家会立即注意到哪些我忽视的东西？请按重要程度排列列出这些盲点。
```

### 9️⃣ 二阶思维者
```
如果【事件/决策】发生，明显的一阶效应是什么？现在深入一层：这些会引发什么二阶效应？由此又会产生什么三阶效应？用简单的列表绘制出这条连锁反应。标记出大多数人会忽视的效应。
```

### 🔟 问题拆解
```
我在这个问题上卡住了：【描述】。将它分解成尽可能小的子问题。对每个子问题，告诉我：这是我现在可以解决的吗？我应该采取的第一步行动是什么？我缺少什么信息？从最容易的子问题开始，帮我建立解决问题的动力。
```

## 📚 学习 (Prompts 11-14)

### 1️⃣1️⃣ 80/20学习计划
```
我想学习【技能/主题】。我每周有【可用时间】。给我80/20版本：能带来80%实用能力的20%核心概念。按周组织计划。每周包括：学什么、一个练习它的练习，以及如何知道我真正理解了。跳过我暂时不需要的理论。
```

### 1️⃣2️⃣ "解释我的错误"老师
```
我刚刚尝试了【任务/练习】。这是我做的：【粘贴你的作业】。指出每一个错误。对每个错误解释：我做错了什么、为什么是错的、正确的方法是什么，以及一个让我再也不犯同样错误的记忆技巧。要具体，不要只说"做得不错"。
```

### 1️⃣3️⃣ 费曼简化器
```
我以为我理解了【概念】，但不确定。让我向你解释一下，你告诉我我的理解在哪里出了问题：【你的解释】。指出任何漏洞、误解或过度简化。然后用不同的类比重新解释我理解错误的部分。
```

### 1️⃣4️⃣ 面试准备教练
```
我在【时间范围】内有一个【类型】面试，面试的是【公司/职位】。问我他们最可能提问的10个最难的问题。在我回答完每个问题后，给我打1-10分，并告诉我哪里强、哪里弱，以及如何重构我的答案。不要客气，做最严苛的面试官。
```

## 💼 工作 (Prompts 15-18)

### 1️⃣5️⃣ 会议总结器
```
这是我从会议上记的乱七八糟的笔记：【粘贴笔记】。将它们整理成一个结构化摘要，包含四个部分：
1. 已做决定（达成了哪些共识）
2. 行动事项（谁做什么、截止日期）
3. 待解问题（还需要回答哪些问题）
4. 关键语录（任何人说过的重要话）
控制在一页以内。
```

### 1️⃣6️⃣ "三明治反馈"终结者
```
我需要就【问题】向【人/职位】提供反馈。背景：【2-3句话】。请直接写出反馈，不要夸奖三明治，不要用"但是"来软化。要诚实、具体、有建设性。包括：问题是什么（附具体例子）、为什么重要，以及我需要他们具体做出什么改变。语气：尊重但清晰。
```

### 1️⃣7️⃣ 从大脑倾倒中构建SOP
```
我要用杂乱的大脑倾倒方式描述我如何做【任务】。将其整理成清晰、编号的分步标准操作流程，让其他人无需问我问题就能按照执行。如果我的流程有漏洞或不清楚的地方，请标记出来。这是我的大脑倾倒：【粘贴】
```

### 1️⃣8️⃣ 周报生成器
```
这是我本周工作的所有内容：【粘贴笔记/任务/更新】。创建一份周报，包含：
- 已完成（做了什么）
- 进行中（正在进行的及进度）
- 受阻（卡在哪里及原因）
- 下周优先事项（3件最重要的事）
格式化为可直接粘贴到Slack或邮件的形式。300字以内。
```

## 🚀 创作 (Prompts 19-20)

### 1️⃣9️⃣ 商业想法压力测试
```
我有一个商业想法：【用3-4句话描述】。对它进行压力测试。回答：
- 确切的客户是谁，他们有什么痛点？
- 他们现在不用我的产品是如何解决这个问题的？
- 他们为什么会切换？
- 我能在一个周末内构建的最简版本是什么来测试需求？
- 这件事失败的最大原因是什么？
请严苛一些，我需要诚实，不需要鼓励。
```

### 2️⃣0️⃣ "帮我构建这个"
```
我想创建【描述你想要的】。我完全没有技术技能。只使用免费工具，一步一步带我完成构建。每一步：告诉我确切使用什么工具、点击什么、输入什么。假设我什么都不懂。如果某个步骤需要任何编码或技术知识，找一个不需要的替代方案。不要跳过步骤，不要假设我能自己搞清楚任何事情。
```
