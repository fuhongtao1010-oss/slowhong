# 2026-04-10 - Stanley - 天下社群，十死九亡。活下来那个，改名叫门派了

## 摘要
这是一篇把“社群”重新定义为“门派 / 合流体”的长帖。作者认为，大多数社群之所以会死，不是因为运营不努力，而是因为从结构上就被搭成了以群主为中心的广播站。与之相对，真正能长期活下来的组织，应该有门规、传承、身份体系、节奏、机制和复盘，并逐步演化成一种允许多流派并存、靠协议协调协作的“合流体”。

## 原文整理
原帖标题句：
“天下社群，十死九亡。活下来那个，改名叫门派了。”

作者先列出常见的社群死亡症状：
- 群主一消失，群就失去能量来源。
- 只有活动，没有日常关系。
- 成员只有“消费群主内容”这一种参与出口。
- 成员过于同质化，缺少摩擦与新信息。
- 规则混乱或根本没有规则，最后被熵增吞掉。

作者进一步认为，“社群”这个词本身带有中心化广播结构的暗示：
- 群主在中间，成员在四周。
- 信息从中心流向边缘。
- 成员天然被定义成接收者，而不是贡献者。

因此作者提出用“门派”替代“社群”：
- 门派有门规。
- 门派有传承。
- 门派有身份体系。
- 门派不会因为某个中心人物消失就立即瓦解。

在这个基础上，作者提出“合流体”概念：
“一种允许多流派并存、以协议协调共识、在关键时刻统一合流对外输出价值的门派型组织。”

作者把“合流协议”的核心收束为四件事：
- 身份：你是谁，你在当前战役中扮演什么角色。
- 任务：新人进入后第一件事是什么，如何立刻进入参与状态。
- 机制：组织靠流程、文档、提案与决策路径运转，而不是靠群主热情。
- 复盘：每一轮行动后都要记录发生了什么、学到了什么、下一轮怎么改。

作者还补充了几个更底层的“社群死亡原因”：
- 心理所有权从未建立：成员没有真正参与和影响组织走向。
- 节奏不对：只有活动，没有稳定心跳。
- 传播单位选错：过度依赖大号，而不是可信节点。
- 没有共同语言系统：没有自己的词汇、黑话和内部概念，难以形成身份认同。

帖子的后半部分把这些判断落到更具体的方法论上：
- 放弃“社群”一词，重新命名组织。
- 写门规，而不是只写防坏事的群规。
- 设计入门体验，而不仅是行政式入群流程。
- 建立固定节奏，例如双周或每月的同步节点。
- 刻意培养节点型影响者，而不是等待 KOL 自然出现。
- 定期写公开战报和复盘，用真实问题建立公信力。
- 把这套方法论开源，而不是藏在私域或付费墙后面。

我见过太多社群死在半路上。
有的死在第三个月，有的死在第七天，还有的死得很体面——群还在，消息还有人发，但你仔细看，全是广告、全是打卡截图、全是那种你看完之后完全不记得自己看了什么的内容。
那不叫活着，那叫停尸房里还留着暖气。
我自己做社群，也观察别人的社群，看了很多年。某种程度上，我觉得中文互联网在社群这件事上，是有系统性认知错误的。大家以为社群死亡是因为运营不努力，或者定价不对，或者内容不够好。
不是的。
社群死亡，是因为它从一开始就搭错了骨架。

一、社群死亡诊断报告
我先给你一份诊断书，你对照看看，你进过的群是不是这些症状。
症状一：群主消失，群也消失。
这是最常见的死法。整个群的能量来源只有一个人，这个人一旦有事、有情绪、有别的事情要忙，群就立刻进入休眠状态。等群主回来再激活，但每次激活都比上次困难一点，因为成员早就养成了“不活跃”的习惯。
这种群本质上不是一个组织，它是一个以某个人为核心的单向广播站，只不过是双向的形状。
症状二：只有活动，没有日常。
有些群运营得很热闹，三不五时就搞个活动、分享、开麦，但活动一结束，群就立刻沉寂。因为成员之间没有自生长的关系，他们来是为了活动，不是为了彼此。
就像那种只有节假日才回家的人，一年见几次面，叫不上来名字，但逢年过节会抢红包表示存在感。
症状三：进群容易，出口太窄。
一个健康的社群应该有多种角色，有人贡献内容，有人推广传播，有人做连接，有人做维护。但大部分群的“出口”只有一个：消费群主的内容。
消费不等于参与，消费只是一种很低层级的互动，它不会产生归属感，也不会产生价值共创。
症状四：同质化太严重。
所有人背景差不多、认知差不多、问题差不多。这种群表面上很和谐，但它没有摩擦，没有摩擦就没有火花，没有火花就没有能量。
最有价值的社群，往往是那种让你既能找到同路人、又能遇见完全不同的人的地方。你在一个充满镜子的房间里待久了，只会越来越自恋，但你见不到新东西。
症状五：规则混乱，或者没有规则。
没有规则的群不是“自由”，是在等待熵增把它吃掉。很多群主害怕立规矩，觉得一立规矩就会吓跑人。结果是什么？广告越来越多，质量越来越低，真正有价值的人越来越安静，最后悄悄退群，没有一点声音。
那些最安静退群的人，往往是你最不想失去的人。
这五种症状，加在一起，就是我们常说的“社群死亡”。
但我还没说最根本的问题。

二、“社群”这个词，从一开始就是个坑
我认为中文互联网里“社群”这个词本身就是个问题。
这个词在过去十年里被用烂了。做课程的叫社群，卖代餐的叫社群，搞直播的叫社群，找同类的也叫社群，帮人买保险的还是叫社群。
一个词可以用来描述这么多完全不同的东西，它就已经失去意义了。
更严重的是，“社群”这个词里带着一种思维定式：群主在中间，成员在四周，信息从中间流向四周。这是一种中心化的广播结构，它天然地把成员定位成“接收者”，而不是“贡献者”。
接收者不会产生归属感。接收者当他接不到想要的信号的时候，就换台。
所以当我们谈“如何运营社群”的时候，我们在做的其实是在一个天然不利于长期运营的框架里修修补补。
就像在一辆没有发动机的车里装各种配件，然后问为什么它跑不起来。

三、真正死不了的东西，叫门派，不叫社群
我用一个词替代“社群”：门派。
门派和社群最本质的区别是：门派有门规，有传承，有身份体系，有比中心人物更大的存在理由。
门派可以换掌门，门派不会因为某个人消失就消失，因为真正撑着它的是一套价值观和行动框架，而不是某个具体的人。
金庸写了那么多武侠，他笔下的门派，有的掌门死了好几任，但门还在。因为有《九阴真经》，有降龙十八掌，有一套被传承和认同的东西。
反过来，那些短暂崛起然后消失的帮派，基本上都是围着某个人转的。那个人一死，烟消云散。
我在做“Stanley和他的朋友们”的时候，有个核心判断就是：这不是一个以我为中心的粉丝群。它要做的，是一个有自己运行逻辑的系统。
我称这种组织形态为合流体。

四、什么是合流体
合流体的定义：一种允许多流派并存、以协议协调共识、在关键时刻统一合流对外输出价值的门派型组织。
听起来很学术？我翻译一下。
你有你的打法，我有我的打法，他有他的打法。我们不要求所有人都说同样的话、走同样的路、用同样的工具。但当需要统一行动的时候，我们能把力气拧在一起。
这就是合流。
为什么叫“体”不叫“派”？因为“体”更开放，它暗示的是一个可以容纳异质性的结构，而不是封闭的宗派。合流体允许内部有争论，允许不同声音，允许你走自己的路——但它有一套大家共同认可的底层规则。
这套规则，我叫它合流协议。
合流协议的核心只有四件事：身份、任务、机制、复盘。
身份：你进来是谁？你在这里扮演什么角色？你是发起者、连接者、输出者，还是维护者？角色不是固定的，但在某个战役里，你的角色必须清晰。
任务：你进来之后，第一件事是什么？最怕的是进群之后什么都没有发生。人的注意力是有惰性的，如果进群的第一感受是“等一等，这里有什么”，等不了多久，他就不等了。
机制：这个组织靠什么运转？靠群主的热情？不行，太脆弱。靠成员的自觉？也不行，自觉是世界上最不靠谱的燃料。靠机制，靠流程，靠版本化的文档，靠清晰的提案和决策路径。
复盘：每一次战役结束之后，发生了什么？总结了什么？下一轮能改进什么？没有复盘的组织不会成长，只会重复犯同样的错误。
这四件事，缺一个，都会出问题。大部分社群死亡，都可以追溯到其中某一个的缺失。

五、为什么大部分社群研究社群运营，却不研究为什么社群会死
这是一个很有意思的现象。
互联网上关于“如何做社群”的内容，汗牛充栋。什么钩子设计、裂变机制、周复盘框架、引流话术，写得非常详细，非常有体系。
但几乎没有人认真研究过：社群为什么会死？
不是运营层面的死，不是“群主不努力”那种解释，而是更底层的原因。
我做了一些观察和总结，以下是社群死亡的几个深层原因。
原因一：心理所有权从未建立。
人会为自己真正觉得有参与感的东西持续付出，但前提是要有“这是我的一部分”的感受。
这个感受，心理学上叫心理所有权，它不是靠口号建立的，不是靠“大家共建”的宣言建立的，而是靠真实的参与、真实的贡献、真实的被看见来建立的。
大部分社群里，成员的参与感来自“我付了钱”或者“我发了消息”，这两种都不构成心理所有权。心理所有权需要你真正地影响了这个地方的走向，你的提案被采纳过，你的问题引发了讨论，你的存在被其他成员认可。
如果一个成员在你的群里消失了，有人会发现吗？
如果没有，这个群对他而言就没有心理所有权，他离开只是时间问题。
原因二：节奏不对。
所有有生命力的组织都有节奏。军队有作息，学校有学期，寺庙有课诵，公司有财年。
节奏不是仪式感，它是一个组织内部的心跳。
大部分社群没有节奏。它们有活动，但没有规律的节奏。活动是随机的，高峰和低谷都无法预测，成员不知道下一次能量爆发什么时候来，所以他们不会等，慢慢地就脱离了。
我在“合流体”里推的一个东西叫合流日，就是固定的、周期性的线上或线下同步仪式。
不需要多宏大，可以就是每个月某天的一次集体回顾和讨论。但它必须固定，必须有人认真对待，必须产生可以被记录下来的东西。
心跳不能停。
原因三：传播单位选错了。
很多社群依赖“大号转发”来做传播，这是一个越来越贵、越来越没效率的方式。
真正有穿透力的传播，不来自最高处的大喇叭，而来自具体关系链里的可信节点。
你的朋友跟你推荐一个东西，和你刷到一个百万粉丝博主推荐同一个东西，你更信哪个？
大部分人，更信朋友。因为朋友和你有真实的关系，他不会无缘无故推荐一个烂东西给你，他知道你的具体情况，他的推荐有针对性。
这就是为什么“节点型影响者”比“素人KOL”更有效。
节点型影响者不需要粉丝很多，他只需要在他的关系链里被信任。一个拥有两千个真实关系、每一个关系都足够深的人，比一个拥有十万粉丝但跟所有人都只是算法上的连接的人，传播效率要高得多。
我一直在做的，不是孵化KOL，而是培养可信节点。
原因四：没有共同的语言系统。
最容易被忽视的一点。
一个真正运转良好的组织，一定有自己的词汇。有自己的黑话，有自己的缩写，有自己对某些概念的独特定义。
这些词汇不只是方便沟通，它们是身份认同的载体。当你说出这些词、被别人理解的时候，你感受到的是“我属于这里”。
大部分社群没有自己的语言系统。他们用的是全互联网通用的词汇，所以他们也建立不了独特的身份感。
我花了很长时间在给“合流体”定名字、定词汇。合流者、合流战役、合流节点、合流日——这些词听起来可能有点奇怪，但正是因为它们奇怪，它们才有力量。你不会在别的地方听到这些词，当你开口用这些词的时候，你就已经在传递一个信号：我是这里的人。

六、一条帖子，千粉。每周百万。这不是天赋，是协议
现在说点具体的。
很多人觉得在 X 上涨粉，靠的是天赋、靠的是运气、靠的是算法的随机眷顾。
这种想法让我很难受，因为它把一件有规律可循的事情变成了玄学。
我的首席技术官，账号“AI最严厉的父亲”，在 2026 年 3 月底经历了一次大规模的病毒式传播，涨粉速度超过了所有人之前预期。有人问他是怎么做到的，他的回答让很多人失望：他只是系统性地执行了一套协议，然后在对的时间发出了对的内容。
没有秘密，只有协议。
关于千粉一条帖子：
一条帖子能带来一千粉丝，需要满足三个条件。
第一，这条帖子必须触发一种具体的情绪，不是模糊的“有共鸣”，而是精准的某一种——愤怒、惊讶、被戳中的那种痒、突然清醒了的感觉。
第二，这条帖子必须让看到它的人觉得“我必须转发，不转发显得我没看懂”或者“我必须转发，不转发对不起我的关注者”。这两种感受都会驱动转发，而转发才是增粉的真实引擎，不是点赞。
第三，这条帖子必须在一个你已经建立了一定公信力的领域发出。零公信力发一条天才帖子，也可能没有回响。因为算法和人类都有一种共同的偏见：他们更容易相信有历史积累的声音。
这三个条件，没有一个靠天赋，全部靠系统性的工作。
关于每周百万阅读：
这是一个可以工程化的目标，不是一个玄学目标。
我的做法是：把内容分成两种，一种是“定期输出”的日常内容，维持账号的基础体温；另一种是“蓄力爆发”的重磅内容，专门设计来引发大规模传播。
日常内容的密度要高，哪怕是一句话、一个观察，每天都有。这是在给算法喂食，也是在给关注者保持存在感。
重磅内容不能每天发，每天发重磅就没有重磅了。但每周至少有一条是经过认真设计的，从选题、角度、开头到结尾，都经过考量。
重磅内容的判断标准只有一个：我发完这条帖子之后，我自己会不会觉得“这条很好”？
不是问别人，是问自己。因为你是这个领域里最了解你的受众的人，如果你自己看完都觉得平平，你的受众十有八九也会这么觉得。

七、开源这套打法
我要把以上的东西整理成一套可以复用的框架，任何人都可以拿去用。
这就是所谓的“开源社群打法”，也是“合流协议 1.0”的草稿版本。
第一步：先给自己的组织一个新名字，把“社群”这个词扔掉。
不是为了好玩，是因为你用什么词描述它，就会用什么框架去建立它。
叫什么不重要，重要的是这个名字要能承载你对这个组织的真实愿景。如果你只是想做一个信息共享群，那就叫群，不要包装成社群；如果你想做一个真正的协作体，那给它一个配得上你野心的名字。
第二步：写门规，不要写群规。
群规是防止人做坏事的，门规是定义这个组织是什么的。
门规只需要三到五条，每一条都要能回答“为什么我们在一起”这个问题。
比如我的第一条：我们在一起，是因为我们都相信 AI 时代需要真实的实践者，不是旁观者。
这一条话，已经过滤掉了大量的人。过滤是好事，不是坏事。
第三步：设计入门体验，不是入群流程。
入群流程是行政的：填表、审核、进群、看置顶。
入门体验是有温度的：你进来第一天，有什么事情发生？有没有人欢迎你？有没有第一个任务让你立刻有参与感？
如果一个人进群的第一感受是“哦，我进来了，那然后呢”，你就已经输了一半。
第四步：建立最小可行的节奏系统。
周期性的同步节点，每月或每双周一次，不要太频繁，频繁就变成负担。
每次同步节点必须产出一个可以被记录下来的东西：一份简短的战报、一个复盘文档、一个决定。
记录不是为了档案，是为了让成员感受到时间在流动，感受到这个组织在成长。
第五步：刻意培养节点，不要等待 KOL 自动冒出来。
节点型影响者不会自然涌现，你需要识别他们、支持他们、给他们资源和空间。
识别的标准不是粉丝数量，而是他们在真实关系链里的影响力，以及他们对你们共同价值观的理解程度。
当这些节点开始在他们各自的关系链里转述你们的理念，这个组织就开始有了不依赖中心节点的生命力。
第六步：定期写战报，公开复盘，不只是在内部。
这是最违反直觉的一条，但也是最重要的一条。
把你们做了什么、遇到了什么问题、解决得好不好，公开地写出来，让外界看见。
很多人害怕暴露问题，觉得这会削弱外界对自己的信任。但现实恰好相反：真实的复盘，特别是那种“我们搞砸了，这是我们学到的”的复盘，是建立长期公信力最快的方式。
因为它证明了你是一个在真实做事的人，不是一个在包装自己的人。

八、关于为什么要开源这套东西
有人问我，你把打法公开了，别人照着做，你不担心被超越吗？
我有一个回答，这个回答也是我自己真正相信的：
如果你的核心竞争力是“只有你知道某个秘密”，那这个竞争力太脆弱了。秘密是会泄露的，方法是会被复制的，技术是会被赶上的。
真正难以被复制的，是你的执行力、你的关系网络、你积累的历史信任、以及你面对真实复杂情况时的判断力。
这些东西，你写成文章、开源出去，别人拿去读了，还是照样无法复制。因为它们不在文字里，它们在你的骨子里。
所以开源不是大方，是因为你足够清楚自己的真实竞争优势不在秘密里。
还有另一层：开源本身也是一种传播方式。
当你的方法论被引用、被讨论、被挑战、被改进，你的影响力就在这个过程里扩散了出去。闭源的方法论死在你的私域里，开源的方法论会找到自己的路。
我希望有一天有人能对我说：“我用了你那套合流协议，但我做了这些改动。”
那说明这套东西真的活了。

九、最后说几句不那么好听的话
做社群——或者说做门派——这件事，没有捷径。
所有那些声称“七天打造百人高质量社群”的课，大概率在教你的是如何快速收一批钱，而不是如何建立一个真正可以长期运转的组织。
这两件事，目标不同，方法不同，结果也完全不同。
第一种快，第二种慢。
我做“Stanley和他的朋友们”三个多月，有过高光，有过沉寂，有过成员离开，有过争论，有过我自己也不知道该往哪走的时候。
但它还在。而且它每过一段时间，都会出现让我意外的东西——一个我没有预料到的人、一次我没有设计的连接、一个从内部自发生长出来的想法。
这才是一个组织真正活着的标志：它在产生你没有设计过的东西。
你做了设计，但结果超出了设计。
那些超出的部分，才是它真正的生命。
关于“合流协议”的完整版本，以及“合流体”组织形态的详细设计文档，我会在接下来的时间里陆续公开在 stanley.team。
不是以课程的形式，而是以开源文档的形式。
因为我相信，一套真正经得起考验的打法，应该能够在公开的环境里被讨论、被质疑、被改进，而不是只在付费墙后面存活。
你来读，你来用，你来改，你来告诉我哪里不对。
这才是合流。
