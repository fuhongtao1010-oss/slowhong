## 摘要

作者以"日销9900+"的从业者身份，揭露了一批推特博主利用"卖Token暴富"话题割韭菜的套路：他们晒虚假数据、用传销式话术招募代理、层层抽成。核心论点是——真正赚钱的是卖教程和代理费的人，而非底层代理。作者随后提供了技术教程：自己搭建 Sub2API 服务，独立做中转站生意，并列出了具体的服务器推荐（ RackNerd）和定价策略。最后列出该生意的五大风险：封号、客服成本、价格战无胜算、合规收紧、无技术壁垒。

### 核心观点
- 这类帖子的本质是卖"你也行"的幻觉，底层代理是被层层收割的韭菜
- 自己搭 Sub2API，数据在己、定价自主、利润独享，才是正经做生意

### 技术教程（Sub2API）
- 项目地址：https://github.com/Wei-Shaw/sub2api（6000+ star）
- 安装方式：`curl -sSL https://raw.githubusercontent.com/Wei-Shaw/sub2api/main/deploy/install.sh | sudo bash` 或 Docker 部署
- 利润估算：Claude Max $200/月 → 反代价值约$2400 API调用，12倍杠杆，实际月赚几百至一千
- 推荐服务器：RackNerd $10.98/年（约¥80）或 $18.29/年（2G内存）
- 定价参考：包月卡200-300元，小红书+闲鱼获客

### 五大风险
1. **封号**：Anthropic封杀手段升级（指纹追踪、Machine ID、GA3），多号轮换只能续命
2. **客服成本**：个人客户售后消耗大量时间，建议用FAQ和自助系统自动化
3. **价格战**：灰色渠道压到¥1/刀，合规渠道最低¥6.5/刀，无法竞争
4. **合规**：《生成式AI服务管理暂行办法》+国安部2026年4月安全警示，窗口收紧
5. **无技术壁垒**：Sub2API开源，护城河只有客户信任和口碑

### 结论对比
自己搭 vs 跟代理体系混：数据自主/定价自主/利润独享 vs 数据在别人服务器/被抽成/风险共担
