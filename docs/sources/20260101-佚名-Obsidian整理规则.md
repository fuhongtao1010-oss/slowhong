# Obsidian 整理规则

这份规则用于减少后续继续变乱。

## 放置原则

- `临时/`：刚收集、刚抓取、还没判断价值的内容
- `记录/`：个人过程记录、复盘、状态日志
- `内容创作/`：写作方法、选题、素材、创作流程
- `AI工具/`：最新技能清单、资讯摘录、尚未沉淀的 AI 观察
- `03-知识/工具/`：网络、代理、VPS、部署、连接相关内容
- `副业机会/`：偏项目、变现、赚钱机会
- `副业新闻/`：资讯流、日报、抓取结果
- `06-资源/`：模板、配置、MOC、索引、通用参考

## 文件命名建议

- 主题笔记：`主题名.md`
- 汇总页：`00-主题汇总.md` 或 `主题MOC.md`
- 抓取内容：`来源-主题-日期.md`
- 日报内容：`主题日报-YYYY-MM-DD.md`

## 每周 10 分钟整理动作

1. 打开 `临时/`，删除无价值内容
2. 把值得保留的内容移到正式目录
3. 给新增主题补一个汇总页
4. 重复内容保留一份，其他笔记互相链接

## 当前不建议立即做的动作

- 不建议一次性重命名全部目录
- 不建议不用链接更新工具就大规模移动笔记
- 不建议改 `.obsidian/` 配置文件

## 当前 Vault 的现实策略

先“加入口”，再“慢慢归并”。

也就是：

- 先用导航页统一入口
- 先用 MOC 把内容串起来
- 后续再逐步把 `AI工具` 里有长期价值的内容沉到 `03-知识`
