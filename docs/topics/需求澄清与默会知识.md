# 需求澄清与默会知识

## 核心问题
当一个想法还很模糊时，怎样把它从“我大概知道我要什么”推进到“我能说清楚、能执行、能交付”？当语言本身不够时，又该如何继续澄清？

- 苏格拉底
- 波兰尼

## 当前理解
这条方法链可以先粗分成三层：

1. 维特根斯坦层：先发现模糊在哪里，把一句大而化之的话拆成更小的问题单元。
2. 苏格拉底层：继续追问，把能说清楚的部分挖出来，包括目标、标准、边界和真正卡住的点。
3. 波兰尼层：承认有些判断并不能完整说出来，此时要改用示范、反例和历史行为去提取偏好。

它的意义不只在 prompt 写法，而在更广义的沟通：需求讨论、决策澄清、带团队、交付对齐、审美偏好提取，都会遇到“知道但说不清”的问题。
