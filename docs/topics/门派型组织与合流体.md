# 门派型组织与合流体

## 核心问题
- 为什么大多数“社群”会死，而少数组织能跨越创始人中心继续活下去？
- 一个组织要如何从“群主广播站”转成“有传承、有机制、有身份体系的门派型结构”？
- 当成员异质性很强时，靠什么把不同流派重新拧成协作力量？

## 子主题
- 社群死亡的结构性原因
- 门规、传承、身份体系
- 合流协议：身份、任务、机制、复盘
- 节奏系统与固定同步节点
- 节点型影响者与关系链传播
- 组织内部的共同语言系统

## 当前理解
- “社群”这个词常常默认的是中心化广播模型：一个人输出，其他人接收。这种结构很适合短期活跃，不适合长期自生长。
- “门派型组织”强调的不是热闹，而是可持续性：成员是否知道自己是谁、为什么在这里、如何参与、如何传承。
- “合流体”比传统门派再往前一步：它不是要求所有人统一风格，而是允许多流派并存，再通过协议在关键节点达成协作。
- 如果一个组织的参与出口只有“消费内容”，成员就很难产生心理所有权；没有心理所有权，就很难形成长期留存和自发贡献。
- 一个真正活着的组织，往往会长出创始人没有预先设计过的连接、行动和产出。

## 值得继续观察的问题
- “门派 / 合流体”是否只是更强叙事包装，还是确实对应不同的组织操作系统？
- 门规与协议如何避免变成空话，而能落实为新成员体验、文档、任务、节奏与复盘机制？
- 什么样的组织规模还适合“门派型”运作，什么规模之后需要更正式的治理结构？
- 这种方法论更适用于内容型组织、学习型组织，还是也适用于创业团队 / 开源社区？
